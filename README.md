## Why This Exists

AI coding agents are powerful but make predictable mistakes:

```
 вқҢ Over-engineering      вҶ’ building abstractions nobody asked for
 вқҢ Silent assumptions    вҶ’ picking interpretations without asking
 вқҢ Scope creep           вҶ’ refactoring adjacent code during changes
 вқҢ Skipping understandingвҶ’ jumping straight to code
 вқҢ Context loss          вҶ’ forgetting everything between sessions
```

**Dev Methodology** fixes all five вҖ” with one workflow.

## What Each Phase Does

### Phase 1: Understand рҹӨ”
```
 Ask questions вҶ’ Identify problem вҶ’ Confirm with user
 в””в”Җ рҹ“ҡ Create knowledge/KNOWLEDGE.md with Vision section
```

### Phase 2: Spec рҹ“қ
```
 Write minimal spec вҶ’ Show in chunks вҶ’ Get approval
 в””в”Җ рҹ“ҡ Add Architecture + Decisions sections
```

### Phase 3: Plan рҹ“Ӣ
```
 Break into tasks вҶ’ Mark dependencies вҶ’ Identify test cases
 в””в”Җ рҹ“ҡ Add Progress checklist (all unchecked)
```

### Phase 4: Implement вҡЎ
```
 Check existing вҶ’ Implement minimal вҶ’ Test вҶ’ Commit
 в””в”Җ рҹ“ҡ Update Progress, add Learnings + Files
```

### Phase 5: Test вң…
```
 Run all tests вҶ’ Manual smoke test вҶ’ Report pass/fail
 в””в”Җ рҹ“ҡ Update Progress with test results
```

### Phase 6: Review рҹҺҜ
```
 Summarize вҶ’ Count LOC вҶ’ Self-critique checklist
 в””в”Җ рҹ“ҡ Add Next section with follow-ups
```

## Core Principles

### рҹҺҜ Minimal Code (Ponytail)
```
 Before writing ANY code:
   1. Does this need to exist at all?
   2. Does a library/tool already solve this?
   3. Does the platform have it natively?
   4. Can it be one line?
   5. Only then write code.
```

### рҹ”Ә Surgical Changes (Karpathy)
```
 When editing existing code:
   вң— Don't "improve" adjacent code
   вң— Don't refactor things that aren't broken
   вң“ Match existing style
   вң“ Every changed line traces to user's request
```

### рҹҡҰ Phase Gates (Superpowers)
```
 Each phase requires user approval.
 No skipping ahead (without a warning).
```
